и“қзүҷ(BLE)еҚҸи®®еҲҶжһҗ- BLEең°еқҖзұ»еһӢ

еҸ‘иЎЁдәҺ 2020-12-25 | еҲҶзұ»дәҺ жҠҖжңҜ - BLE - BLEеҚҸи®®еҲҶжһҗ
йҳ…иҜ»ж—¶й•ҝ ≈

BLE е’Ң WiFi зҡ„ MAC ең°еқҖиҝҳжҳҜдёҚеҗҢзҡ„пјҢиҝҷйҮҢз®ҖеҚ•жҖ»з»“дёҖдёӢгҖӮ

д№ҹеҸҜд»ҘеңЁBluetooth Core specification v4.2, Vol 6, Part B, 1.3 DEVICE ADDRESS жҲ–иҖ… Bluetooth Core specification v5.0, Vol 6, Part B, 1.3 DEVICE ADDRESS

1. Public Device Address

еңЁз»Ҹе…ёи“қзүҷдёӯ(Bluetooth Core specification v4.0 д№ӢеүҚ),
иҝҷдёӘе…¬е…ұи®ҫеӨҮең°еқҖдёҺ WiFi жЁЎз»„дёӯзҡ„ MAC ең°еқҖдёҖж ·пјҢйғҪжҳҜйңҖиҰҒеҗ‘ IEEE з”іиҜ·е№¶дё”иҙӯд№°пјҢ еҪ“з„¶дәҶ WIFi зҡ„иҝҷдёӘ MAC ең°еқҖжҳҜз”ЁеңЁ TCP/IP зҪ‘з»ңдёӯзҡ„пјҢи“қзүҷзҡ„иҝҷдёӘ е…¬е…ұи®ҫеӨҮең°еқҖжҳҜеңЁи“қзүҷдёӯзҡ„гҖӮ
дҪҶжҳҜд»–д»¬йғҪжңүдёҖдәӣдёҖж ·зҡ„еұһжҖ§,е°ұжҳҜе”ҜдёҖжҖ§гҖӮ

еҶҚиҖ…пјҢе…¶и®ҫеӨҮең°еқҖжҳҜдёҖдёӘ 48bits зҡ„ж•°еӯ—пјҢз§°дҪң 48-bit universal LAN MAC address гҖӮ

и“қзүҷеҚҸи®®ж Ҳд№ҹйҒөеҫӘдәҶиҝҷз§Қең°еқҖеҲҶй…Қж–№ејҸгҖӮз”ұ 24-bit зҡ„ company_id е’Ң 24-bit зҡ„ company_assigned з»„жҲҗгҖӮ

2. Random Device Address

дҪҶжҳҜеҲ°дәҶи“қзүҷдҪҺеҠҹиҖ— BLE ж—¶д»Ј, еңЁ Public Device Address еҹәзЎҖд№ӢдёҠеўһеҠ дәҶйҡҸжңәи®ҫеӨҮең°еқҖ Random Device Address гҖӮ

еӨ§иҮҙзҡ„еҺҹеӣ еҰӮдёӢпјҡ

  1. еҗ‘ IEEE з”іиҜ·е’Ңз®ЎзҗҶжҜ”иҫғз№ҒзҗҗпјҢиҖҢдё”иҙӯд№°ең°еқҖжҳҜйңҖиҰҒиҠұй’ұзҡ„
  2. жң¬иә«и“қзүҷеҚҸи®®и§„иҢғдёӯеҫҲеӨ§дёҖйғЁеҲҶеә”з”ЁеңәжҷҜжҳҜе№ҝж’ӯпјҢжүҖд»Ҙдјҡеӣ дёәжҡҙйңІи®ҫеӨҮең°еқҖиҖҢжҡҙйңІжӣҙеӨҡзҡ„дҝЎжҒҜгҖӮ

еҹәдәҺжӯӨпјҢд»Һ Bluetooth Core specification v4.0 ејҖе§ӢпјҢи“қзүҷеҚҸи®®ж ҲеўһеҠ дәҶдёҖз§ҚеҚҸи®®пјҢж”ҜжҢҒйҡҸжңәи®ҫеӨҮең°еқҖпјҢиҝҷз§Қи®ҫеӨҮең°еқҖдёҚжҳҜеӣәе®ҡеҲҶй…Қзҡ„пјҢиҖҢжҳҜжңүи®ҫеӨҮеҗҜеҠЁеҗҺйҡҸжңәз”ҹжҲҗзҡ„гҖӮ

еңЁжӯӨеҹәзЎҖд№ӢдёҠпјҢеҸҲеҸҜд»Ҙе°Ҷи®ҫеӨҮең°еқҖеҲҶдёәйқҷжҖҒи®ҫеӨҮең°еқҖе’Ңз§ҒеҜҶи®ҫеӨҮең°еқҖгҖӮ

2.1 Static Device Address

Static Device Address жҳҜи®ҫеӨҮеңЁдёҠз”өж—¶йҡҸжңәз”ҹжҲҗзҡ„ең°еқҖгҖӮ жҜ”еҰӮзҺ°еңЁ Nordic й»ҳи®ӨйғҪжҳҜдҪҝз”ЁиҝҷдёӘйқҷжҖҒең°еқҖзҡ„гҖӮ
зӣёжҜ”иҫғ Public Device Address иҖҢиЁҖпјҢеҫҲеҘҪзҡ„и§ЈеҶідәҶз”іиҜ·е’Ңз»ҙжҠӨз®ЎзҗҶ Public Device Address жүҖеёҰжқҘзҡ„й—®йўҳгҖӮ

йңҖиҰҒжіЁж„Ҹзҡ„ең°ж–№жҳҜпјҡ

  1. жңҖй«ҳзҡ„дёӨдёӘ bit дҪҚдёҚеҸҜдҝ®ж”№дё”дёә 11
  2. е…¶дҪҷеү©дҪҷзҡ„ 46bits

2.2 Private Device Address

Private Device Address з§ҒеҜҶи®ҫеӨҮең°еқҖдҪҝз”ЁйҡҸжңәз”ҹжҲҗзҡ„ж–№ејҸгҖӮеҸҜд»Ҙе®ҡж—¶жӣҙж–°е’Ңең°еқҖеҠ еҜҶгҖӮиҝҷж ·д№ҹе°ұжҸҗй«ҳдәҶи“қзүҷең°еқҖзҡ„еҸҜйқ жҖ§е’Ңе®үе…ЁжҖ§гҖӮ

ж №жҚ®ең°еқҖжҳҜеҗҰеҠ еҜҶпјҢеҸҜд»Ҙе°Ҷ Private Device Address еҲҶдёә Non-resolvable Private Address е’Ң Resolvable Pricvate Address гҖӮ

2.2.1 Non-resolvate Private Address

Non-resolvable private address дёҚеҸҜи§Јжһҗз§ҒеҜҶең°еқҖе’Ң Static Device Address зұ»дјјпјҢдёҚеҗҢд№ӢеӨ„еңЁдәҺпјҢNon-resolvable private address дјҡе®ҡж—¶жӣҙж–°гҖӮжӣҙж–°зҡ„е‘Ёжңҹз§°жҳҜз”ұ GAP и§„е®ҡзҡ„пјҢз§°дҪң T_GAP пјҢе»әи®®еҖјжҳҜ15еҲҶй’ҹгҖӮ

дёҚеҸҜи§Јжһҗз§ҒеҜҶең°еқҖзҡ„зү№еҫҒеҰӮдёӢпјҡ

  • жңҖй«ҳзҡ„дёӨдёӘ bit дҪҚйғҪдёә1
  • еү©дҪҷзҡ„ 46 bits жҳҜдёҖдёӘйҡҸжңәж•°пјҢиҮіе°‘дёҖдҪҚжҳҜ0дё”иҮіе°‘дёҖдҪҚжҳҜ1
  • д»Ҙ T_GAP дёәе‘ЁжңҹпјҢе®ҡж—¶жӣҙж–°

дҪҶжҳҜпјҢжҲ‘и§үеҫ—иҝҷзұ»ең°еқҖеә”иҜҘдёҚеёёз”ЁжҲ–иҖ…еңЁзү№ж®ҠеңәжҷҜдёӢдҪҝз”ЁпјҢеӣ дёәең°еқҖиҖҒжҳҜеҸҳжқҘеҸҳеҺ»дёҚжҳҜеқ‘йҳҹеҸӢеҗ—пјҹ

2.2.2 Resolvate Private Address

иҰҒз”ҹжҲҗдёҖдёӘеҸҜи§Јжһҗз§Ғжңүең°еқҖResolvate Private Address(з®Җз§° RPA)пјҢи®ҫеӨҮеҝ…йЎ»е…·еӨҮд»ҘдёӢдёӨз§ҚжқЎд»¶д№ӢдёҖ:
Local Identity Resolving Key жң¬ең°иә«д»Ҫи§ЈжһҗеҜҶй’ҘжҲ– Peer Identity Resolving Key еҜ№зӯүиә«д»Ҫи§ЈжһҗеҜҶй’Ҙ,иҝҷдёӨдёӘйғҪз®Җз§° IRK гҖӮ

еҸҜи§Јжһҗз§Ғжңүең°еқҖз”ұ IRK е’ҢдёҖдёӘйҡҸжңәз”ҹжҲҗ24дҪҚж•°з”ҹжҲҗзҡ„гҖӮйҡҸжңәж•°иў«з§°дёә prand пјҢе№¶еә”жҳҜж»Ўи¶ід»ҘдёӢиҰҒжұӮгҖӮ

  • Prand зҡ„дёӨдёӘжңҖй«ҳзҡ„дҪҚеә”иҜҘжҳҜ вҖң10вҖқ
  • prand зҡ„йҡҸжңәйғЁеҲҶдёӯиҮіе°‘жңүдёҖдёӘдҪҚжҳҜ вҖң0вҖқ
  • prand зҡ„йҡҸжңәйғЁеҲҶдёӯиҮіе°‘жңүдёҖдёӘдҪҚжҳҜ вҖң1вҖқ

еҸҜи§Јжһҗз§Ғжңүең°еқҖзҡ„зү№еҫҒжҖ»з»“еҰӮдёӢпјҡ

  1. з”ұдёӨйғЁеҲҶз»„жҲҗпјҡ
    • й«ҳдҪҚ 24bits жҳҜйҡҸжңәж•°йғЁеҲҶпјҢе…¶дёӯжңҖй«ҳдёӨдёӘ bit дёәвҖң10вҖқпјҢз”ЁдәҺж ҮиҜҶең°еқҖзұ»еһӢ
    • дҪҺдҪҚ 24bits жҳҜйҡҸжңәж•°е’Ң IRK з»ҸиҝҮ hash иҝҗз®—еҫ—еҲ°зҡ„ hash еҖјпјҢиҝҗз®—зҡ„е…¬ејҸдёә $hash = ah(IRK, prand)$
  2. еҪ“еҜ№з«Ҝ BLE и®ҫеӨҮжү«жҸҸеҲ°иҜҘзұ»еһӢзҡ„и“қзүҷең°еқҖеҗҺпјҢдјҡдҪҝз”ЁдҝқеӯҳеңЁжң¬жңәзҡ„ IRK е’ҢиҜҘең°еқҖдёӯзҡ„ prandиҝӣиЎҢеҗҢж ·зҡ„ hash иҝҗз®—пјҢе№¶е°Ҷиҝҗз®—з»“жһңе’Ңең°еқҖдёӯзҡ„ hash еӯ—ж®өжҜ”иҫғпјҢзӣёеҗҢзҡ„ж—¶еҖҷжүҚиҝӣиЎҢеҗҺз»ӯзҡ„ж“ҚдҪңгҖӮиҝҷдёӘиҝҮзЁӢз§°дҪң resolve пјҲи§ЈжһҗпјүгҖӮ
  3. д»Ҙ T_GAP дёәе‘Ёжңҹе®ҡж—¶жӣҙж–°гҖӮеҚідҪҝеңЁе№ҝж’ӯгҖҒжү«жҸҸгҖҒе·ІиҝһжҺҘзӯүиҝҮзЁӢдёӯд№ҹеҸҜиғҪж”№еҸҳең°еқҖгҖӮ
  4. RPA дёҚиғҪеҚ•зӢ¬дҪҝз”Ё,еҰӮжһңдҪҝз”ЁпјҢйңҖиҰҒиҰҒеҗҢж—¶е…·еӨҮ Public Device Address жҲ–иҖ… Static Device Address дёӯзҡ„дёҖз§ҚгҖӮ
  5. RPA е№¶дёҚиғҪдҝқжҠӨе№ҝж’ӯеҢ…зҡ„ж•°жҚ®гҖӮеҰӮжһңжңүж•Ҹж„ҹдҝЎжҒҜпјҢе»әи®®ж”ҫеҲ° scan response packetдёӯгҖӮ
